## 実装順序

1. **Phase 1 → 2 → 3** は直列（基盤のため）。ただし Task 1.3（Readium検証）は Phase 2〜3 と並行可
2. **Phase 4** 内は 4.1 完了後、4.2→4.3 を軸に 4.4/4.5/4.6 は並行可
3. **Phase 5** は 5.1 完了後、5.2〜5.6 を順次（5.3/5.5/5.6 は並行可）
4. **Phase 6** は 6.1 完了後、6.2〜6.4 は並行可
5. **クリティカルパス**: 1.1 → 1.2 → 2.1 → 2.2 → 2.3 → 3.3 → 4.1 → 4.2 → 4.3 → 5.1 → 5.6 → 6.1 → 6.3/6.4 → 7.3

## リスクと対策

- **Readiumの導入・API変化**: Task 1.3 で早期にスパイク検証し、問題があれば代替（自前WebView+epub展開等)の判断を Phase 2 中に行う
- **ePub進捗とKomgaページ番号の対応**: Task 6.3 で positions系APIを実測検証。対応不能なら「completed のみ同期」へフォールバックせず、ユーザーに報告して仕様判断を仰ぐ
- **バックグラウンドDLの完了イベント**: 実機での検証必須（シミュレータは挙動が異なる）
- **実サーバーのレスポンス差異**: Task 2.2 で実サーバーのJSONをフィクスチャ化してテストに使用
