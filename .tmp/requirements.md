# 要件定義書 - iOS向けKomga対応リーダーアプリ「dokusho」

## 1. 目的

セルフホスト型メディアサーバー「Komga」に接続し、iOSデバイス上で書籍（ePub / PDF）を快適に閲覧できるリーダーアプリを開発する。ストリーミング閲覧とダウンロード閲覧の両方に対応し、オフラインでも読書できる体験を提供する。コミック（CBZ/CBR等）は対象外。

## 2. 機能要件

### 2.1 必須機能

#### サーバー接続・認証
- [ ] KomgaサーバーのURL登録（自己署名証明書やローカルネットワーク上のサーバーは不要）
- [ ] 認証対応：Basic認証は不要。APIキー（`X-API-Key`ヘッダー）のみ。
- [ ] セッション管理（`X-Auth-Token` / `KOMGA-SESSION` の再利用によるログイン維持）
- [ ] 認証情報の安全な保存（Keychain使用）

#### ライブラリ閲覧
- [ ] ライブラリ一覧の表示（`GET /api/v1/libraries`）
- [ ] シリーズ一覧の表示（サムネイル付きグリッド表示、ページネーション対応）
- [ ] シリーズ内のブック一覧表示（`GET /api/v1/series/{seriesId}/books`）
- [ ] シリーズ・ブックの検索
- [ ] サムネイル表示（`/thumbnail` エンドポイント）とそのキャッシュ

#### 閲覧方法（2方式）
- [ ] **ストリーミング閲覧**：ページ単位の画像取得（`GET /api/v1/books/{bookId}/pages/{page}`）による逐次閲覧。先読み（プリフェッチ）対応。対象は**PDF**（Komgaがページを画像化して配信）
- [ ] **ダウンロード閲覧**：ブックのファイル一括ダウンロード（`GET /api/v1/books/{bookId}/file`）。対象フォーマットは **ePub** および **PDF**
- [x] **コミック（CBZ/CBR等）は対象外**（ストリーミングもダウンロードも行わない）
- [ ] ダウンロード済みブックのオフライン閲覧
- [ ] ダウンロードの進捗表示・キャンセル・削除

#### リーダー（ビューア）
- [ ] ページめくり：**左送り（右綴じ・マンガ向け）/ 右送り（左綴じ・洋書向け）** の両対応。ユーザーが切替可能
- [ ] Komgaのメタデータ（シリーズの読み方向 `readingDirection`）を初期値として反映
- [ ] 画像ベース閲覧（PDFストリーミング時：ページ画像をスワイプ/タップでめくる）
- [ ] PDF閲覧（PDFKit等によるレンダリング）
- [ ] ePub閲覧（リフロー型テキストのレンダリング）
- [ ] ピンチズーム、ダブルタップズーム
- [ ] ページスライダーによるジャンプ、ページ番号表示

#### キャッシュ
- [ ] ストリーミング時のページ画像キャッシュ（メモリ + ディスク）
- [ ] サムネイルキャッシュ
- [ ] キャッシュ上限設定とクリア機能（設定画面から実行）

#### 読書進捗
- [ ] 読書進捗のサーバー同期（`PATCH /api/v1/books/{bookId}/read-progress`）
- [ ] 続きから読む（サーバー上の進捗の反映）
- [ ] オフライン時はローカルに進捗を保持し、オンライン復帰時に同期

### 2.2 オプション機能

- [x] 【不要】複数サーバーの登録・切替
- [ ] 【必須】見開き表示（2ページ表示、横向き時）
- [x] 【不要】縦スクロール（Webtoon）モード
- [ ] 【必須】コレクション / リードリスト対応
- [ ] 【必須】「次に読む」（Keep Reading / On Deck）表示
- [ ] 【必須】ダークモード連動のリーダー背景色設定
- [ ] 【必須】バックグラウンドダウンロード（URLSession background configuration）
- [ ] 【必須】iPad対応レイアウトの最適化

## 3. 非機能要件

### 3.1 パフォーマンス

- ページめくりは即座（先読み済みページは16ms以内に表示開始）
- ストリーミング時は前後N枚（既定: 3〜5枚）の先読みを行い、めくり待ちを最小化
- 一覧表示はページネーション/遅延読み込みで大規模ライブラリ（数千シリーズ）でも快適に動作
- メモリキャッシュは上限を設け、メモリ警告時に解放

### 3.2 セキュリティ

- 認証情報（パスワード・APIキー・セッショントークン）はKeychainに保存。UserDefaultsへの平文保存禁止
- 通信はHTTPS必須。
- ダウンロードしたファイルはアプリのサンドボックス内に保存

### 3.3 保守性

- API層・キャッシュ層・UI層を分離したアーキテクチャ
- Komga APIクライアントは独立モジュールとしてテスト可能にする
- Swift Concurrency（async/await）ベースの非同期処理で統一

### 3.4 互換性

- iOS 17以上（確定）
- Komga v1.x 系の API v1 に対応
- 画像フォーマット：JPEG / PNG / WebP（Komgaのページ変換パラメータ利用可否を設計時に確認）

## 4. 制約事項

### 4.1 技術的制約

- Swift + SwiftUI をベースとするネイティブiOSアプリ（設計フェーズで確定）
- ePubレンダリングは自前実装が高コストのため、ライブラリ（例：Readium Swift Toolkit）の採用を検討
- Komga API v1 の仕様（https://komga.org/docs/openapi/komga-api/）に準拠
- App Storeガイドライン準拠（自己ホストサーバーへの接続アプリとして問題ない構成にする）

### 4.2 ビジネス制約

- 特になし（個人開発）。まずMVP（接続→一覧→ストリーミング閲覧→DL閲覧）を最短で成立させる

## 5. 成功基準

### 5.1 完了の定義

- [ ] Komgaサーバーにログインし、ライブラリ→シリーズ→ブックを辿って一覧表示できる
- [ ] ePub / PDF をダウンロードし、オフラインで閲覧できる
- [ ] 左送り・右送りを切り替えて自然にページめくりできる
- [ ] ページ画像・サムネイルがキャッシュされ、再表示時に再取得しない
- [ ] 読書進捗がサーバーに同期され、別クライアントと共有される

### 5.2 受け入れテスト

- 実機（iPhone）で自宅のKomgaサーバーに接続し、ePub1冊・PDF1冊を通読できる
- 機内モードでダウンロード済みブックが閲覧できる
- アプリ再起動後もログイン状態・読書進捗が維持される

## 6. 想定されるリスク

| リスク | 対策 |
|---|---|
| ePubレンダリングの実装コストが高い | Readium等の実績あるOSSライブラリを採用。設計フェーズで検証 |
| 大きなPDF/ePubのダウンロードによるストレージ圧迫 | ダウンロード管理画面でサイズ表示・削除を可能にする |
| Komga APIのバージョン差異 | API v1に固定し、サーバーバージョンを接続時に確認 |
| ストリーミング中の回線断 | リトライとエラー表示、読み込み済みキャッシュでの継続 |

## 7. 今後の検討事項（設計フェーズで詳細化）

- アーキテクチャ（SwiftUI + MVVM / TCA等）とモジュール構成
- ePubリーダーライブラリの選定（Readium Swift Toolkit vs 他）
- キャッシュ戦略の詳細（ディスクキャッシュのeviction、上限既定値）
- ダウンロードのバックグラウンド対応の要否（MVPに含めるか）
- ストリーミングとDLの自動切替（ネットワーク状態による）の要否
